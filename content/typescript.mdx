# Conceitos

## O que é?

TypeScript é uma linguagem baseada em JavaScript em que adiciona tipagens em acesso das features mais recentes do JS.

## Mitos e verdades

- TypeScript diminui a produtividade
  - No início a produtividade diminui, por se tratar de uma nova linguagem
- TypeScript é transformar JavaScript em Java ou C#
  - Não. O TypeScript é uma linguagem fracamente tipada.
- O mercado não usa TypeScript?
  - Não. O mercado está se adaptando ao TS.
- TypeScript substitui o JavaScript por completo
  - Não, ele complementa o JS.
- TypeScript atrapalha quem é iniciante.
  - Em partes, apenas cria dificuldades comuns no início do aprendizado.
  - Ajuda com o autocomplete do IntelliSense

## Instalações iniciais do Typescript (configurações de servidor e afins)

Instalação do express:

```shell
$ yarn add express
```

Instalações de dependências de desenvolvimento

```shell
$ yarn add typescript tsconfig-paths @types/express -D
```

## Configuração do Nodemon

Arquivo **`nodemon.json`**

```json
{
  "watch": ["src"],
  "ext": ".ts,.js",
  "ignore": [],
  "exec": "node --inspect -r tsconfig-paths/register -r sucrase/register"
}
```

## Configuração do Typescript

Arquivo **`tsconfig.json`**

```json
{
  "compilerOptions": {
    "target": "es5",
    "module": "commonjs",
    "allowJs": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "strictPropertyInitialization": false,
    "moduleResolution": "node",
    "baseUrl": "./src",
    "allowSyntheticDefaultImports": true,
    "esModuleInterop": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,
    "forceConsistentCasingInFileNames": true,
    "noImplicitAny": true,
    "typeRoots": ["node_modules/@types", "src/@types"],
    "paths": {
      "@config/*": ["config/*"],
      "@services/*": ["services/*"],
      "@schemas/*": ["schemas/*"],
      "@queues/*": ["queues/*"],
      "@utils/*": ["utils/*"]
    }
  },
  "include": ["src", "__tests__"]
}
```

## Configuração do App

Arquivo **`src/app.ts`**

```ts
import express from "express";

const app = express();

app.use(express.json());

export default app;
```

## Configuração do servidor

Arquivo **`src/server.ts`**

```ts
import app from "./app";

app.listen(3333, () => {
  console.log("⚡️ Server listening on http://localhost:3333");
});
```

## Criação de Rotas

No Typescript as rotas se comportam semelhantes ao JS.

Exemplo de um arquivo **`routes.ts`**

```ts
import { Router } from "express";

const routes = Router();

export default routes;
```
